## Pflege-Regeln

1. Diese Datei ist die **primäre Aufgabenquelle**. `docs/tasks.docx` bleibt nur als
   lokaler Eingang für Aufgaben mit privaten Screenshots (gitignored, nie committen);
   solche Aufgaben werden hier ohne private Details gespiegelt.
2. Immer nur **ein** Arbeitspaket gleichzeitig beginnen (oberste offene Aufgabe zuerst,
   sofern Jakob nichts anderes vorgibt).
3. Ein Paket gilt erst als abgeschlossen nach: Umsetzung → Doku → lokalem Test → Commit →
   Push → Live-Prüfung. Erst dann hier umtragen.
4. Diese Datei ist öffentlich (Repo): keine privaten Daten, keine Screenshots, keine internen
   Zugangsdaten.

## Nutzungskontingent-Regel (für KI-Agenten)

Claude/Codex können ihr verbleibendes Nutzungskontingent **nicht programmatisch abfragen** –
die Anzeige gibt es nur in der jeweiligen App-Oberfläche. Deshalb gilt als Arbeitsweise:

- Arbeitspakete klein und in sich abschließbar schneiden (Ziel: jedes Paket in einer
  Sitzung fertig inkl. Push).
- Nach **jedem** abgeschlossenen Schritt diese Datei aktualisieren, damit ein Abbruch
  jederzeit einen sauberen, nachvollziehbaren Stand hinterlässt.
- Wenn ein Limit absehbar ist (z. B. Hinweis von Jakob oder sehr lange Sitzung): kein neues
  Paket mehr beginnen, laufendes Paket fertigstellen oder auf einen konsistenten
  Zwischenstand bringen, committen und den Status hier eintragen.
- Ein abgebrochenes Paket bleibt unter „In Arbeit" stehen, mit kurzer Notiz, was fehlt.
